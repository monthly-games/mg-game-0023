## 2. 코어 루프 (Core Loop)

```
┌─────────────────────────────────────────────────────────┐
│  1. 자원 채집/생산                                       │
│         ↓                                               │
│  2. 기지/시설 건설                                       │
│         ↓                                               │
│  3. 위기/이벤트 대응                                     │
│         ↓                                               │
│  4. 식민지 확장/발전 ─────────────────────→ (1로 반복)    │
└─────────────────────────────────────────────────────────┘
```

### 루프 상세 설명

1. **자원 채집**: 다양한 자원 수집 및 가공
2. **기지 건설**: 시설 배치, 구조 설계
3. **위기 대응**: 자연재해, 외부 위협, 내부 문제 해결
4. **식민지 확장**: 새 지역 개척, 인구 증가

---

## 3. 재미 축 (Fun Pillars)

### 3.1 생존/위기 관리 (Survival)
- **목표**: 위기 상황에서의 긴장감과 극복의 재미
- **핵심 경험**: 위기를 넘겼다! 하는 안도와 성취
- **주요 요소**:
  - 자원 관리
  - 재해 대응
  - 주민 생존
  - 위협 방어

### 3.2 기지/식민지 설계 (Base Design)
- **목표**: 효율적인 기지를 설계하는 전략적 재미
- **핵심 경험**: 이 배치가 최적이야! 하는 발견
- **주요 요소**:
  - 시설 배치 최적화
  - 동선 설계
  - 자원 흐름
  - 방어 구조

### 3.3 장기 목표 (Long-term Goals)
- **목표**: 거대한 목표를 향한 장기적 몰입
- **핵심 경험**: 드디어 이 단계까지 왔다
- **주요 요소**:
  - 기술 트리
  - 확장 마일스톤
  - 엔딩 목표
  - 업적 시스템

## 4. 디자인 레버 (Design Levers)

### 4.1 상호작용 시스템
| 레버 | 설명 |
|------|------|
| 자원/위험/환경 연결 | 자원/위험/환경 요소 간 상호작용 세밀하게 설계 |
| 구현 방향 | 물 부족 → 작물 고사 → 식량 위기 → 주민 불만, 연쇄 반응 시스템 |

### 4.2 이벤트 긴장감
| 레버 | 설명 |
|------|------|
| 위기 이벤트 | 이벤트/위기(폭풍, 괴생물체, 반란 등)로 긴장 유지 |
| 구현 방향 | 랜덤 재해, 주기적 침략, 내부 갈등 이벤트 |

### 4.3 난이도 커스텀
| 레버 | 설명 |
|------|------|
| 고인물 대응 | 난이도 프리셋/커스텀 룰로 고인물 유저 대응 |
| 구현 방향 | 이지/노멀/하드/익스트림, 커스텀 룰(자원 50%, 재해 200% 등) |

## 5. 재미 측정 지표 (KPI)

| 지표 | 목표 | 측정 방법 |
|------|------|-----------|
| 평균 세션 시간 | 30분 이상 | 세션 길이 |
| 기지 생존 일수 | 평균 100일 | 게임 내 경과 시간 |
| 위기 극복률 | 70% 이상 | 위기 발생 → 극복 비율 |
| 확장 진행률 | 월 1단계 | 확장 마일스톤 |
| D7 리텐션 | 25% 이상 | 7일차 잔존율 |

## 6. 리스크 및 대응

| 리스크 | 영향 | 대응 방안 |
|--------|------|-----------|
| 높은 진입 장벽 | 이탈 증가 | 튜토리얼 강화, 가이드 |
| 실패 스트레스 | 좌절감 | 체크포인트, 복구 옵션 |
| 복잡도 | 이해 어려움 | UI/UX 개선, 정보 시각화 |
